NEAT Research Project

From Spring 2014 to Fall 2015, I worked on a research project within the University of Texas's Freshman Research Initiative program to attempt to develop an Artificial Intelligent agent to play and beat the SNES version of the Super Mario Bros. game using my own implementation of the NEAT algorithm. The Neuroevolution of Augmented Topologies (NEAT) algorithm was originally developed by Dr. Stanley for his dissertation and it provided a novel way to evolve a Neural Network to solve a problem by using an idea called speciation - separating neural nets into groups (species) that are similar and restricting which networks they can 'breed' with to only those within their species. Another key aspect of this algorithm is its evolution process includes mutation for both the Network structure and the weights of the individual neurons. Unlike Deep Learning, which creates large, non-changing network structure and then evolves the weights to solve some problem, NEAT begins with a population of the smallest possible network structure and expands it through mutation.

Using Stanley's dissertation as a reference, I implemented my own version of it to be ran on my own implementation of the Super Mario Bros. game. As a part of a voluntary research fellowship, I worked, and for the most part finished, the implementation of NEAT over the summer after my Freshman year. During the next fall semester my colleague, Mark Bakke, joined me on this project and helped with implementing the game and running some of our experiments. Simple Ray Tracing Algorithm

During Spring of 2015, I decided to try to learn more about the field of Computer Graphics, so as an initial project I decided to create implement a Ray Tracing algorithm. Ray Tracing is a technique for simulating the lighting for a scene where every pixel in the image for the scene is calculated by following a ray from that pixel straight into the scene, discovering what objects and where on those objects those rays intercept. From there, more rays are generated to bounce off from that location to simulate refraction and a ray is shot from that location to every light source in the scene to calculate the light for the pixel the initial ray represented based on the distance to the light source and the objects between the ray's origin and the light source.

Level Editor

As a spin off side project to aid in my implementation of Super Mario Bros, I began creating a Level Editor that could be used to make a level for any type of two dimensional game. To accomplish this I created it so that the level designer could assign each entity in the level the exact types representing their variables in source code. Thus, making it so the level editor would be generic enough for a level designer to make a level for any imaginable game. I did not finish all of the functionality for this project that I initially wanted to implement, however, I plan on finishing it in the future.
